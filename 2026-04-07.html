生态竞争

Anthropic 封锁 Claude Code 调用 OpenClaw，OpenAI 立刻开源 codex-plugin-cc 反击

4月4日起，Anthropic 正式禁止 Claude Code 订阅用户用包月额度调用 OpenClaw 等第三方智能体工具，强制切换为按量付费。OpenClaw 创始人（已加入 OpenAI）与 Anthropic 谈判失败，仅争取到延期一周执行。OpenAI 随即在 GitHub 开源 codex-plugin-cc，上线半天获3200+ Stars，让 Claude Code 用户可直接调用 Codex，跨厂商生态卡位战全面升级。

核心信号：AI 巨头之间的竞争已从模型性能之争，演变为生态入口之争——谁控制了开发者工作流的"调用层"，谁就掌握下一代 AI 应用的分发权。这场战争的受益者是开发者，受损者是任何试图"封闭生态"的一方。

博通与谷歌签署至2031年 TPU 长期协议；并为 Anthropic 锁定3.5GW 算力

博通今日向 SEC 提交监管文件，正式宣布与谷歌签署长期合作协议，将为谷歌开发并供应未来几代定制 TPU 及下一代 AI 数据机架网络组件，协议至2031年。同时，博通与 Anthropic 扩大合作，自2027年起为 Anthropic 提供约3.5吉瓦的 TPU 算力支持，消息公布后博通股价盘后涨超3%。算力的长期锁定，正在成为 AI 巨头竞争的新形式。

Meta 宣布开源新一代 AI 模型，Alexandr Wang 入主后首批成果亮相

Meta 正式宣布准备发布新一代 AI 模型，并向开发者提供部分版本开源授权，这是 Scale AI 创始人 Alexandr Wang 出任 Meta AI 负责人后推出的首批模型成果。Meta 表示将保留部分核心组件为专有技术，安全评估完成后逐步开放可修改版本。Meta 的开源策略历来是行业参照系，此次部分开源被解读为在 Llama 路线上的策略收紧。

思考延展 · HRD 视角
INSIGHT 01

Anthropic 封锁 OpenClaw：生态之战的 HR 启示——"平台依赖"是组织风险

Anthropic 的封锁让依赖 Claude Code + OpenClaw 组合的开发者一夜之间被迫切换，这不只是商业事件，也是一个组织风险案例：当团队的核心工作流重度依赖单一外部工具，该工具的策略调整就直接变成组织效率危机。

对 Daisy 的启发：TME 在推广内部 AI 工具时，需要避免让某个团队对单一 AI 工具形成"单点依赖"。HR 可以在 AI 工具培训设计中加入"工具迁移能力"——不只是会用某个工具，而是建立对 AI 工具的通用操作能力，这样即使工具换了，效率不打折。

INSIGHT 02

爱奇艺用"AI 叙事"撬动资本市场：HR 的 AI 故事也需要"对外叙事能力"

爱奇艺赴港上市时把 AI 智能体产品作为核心卖点——这个逻辑不只适用于资本市场，同样适用于 TME 内部：HR 的 AI 工作推进，也需要一套能让管理层、业务部门、员工都能听懂的"AI 叙事"，而不只是内部执行报告。

对 Daisy 的启发：把 AI 能力建设工作"产品化表达"——不说"我们做了AI培训"，而是说"我们为TME构建了AI协作能力基础设施"。这种表达方式在向新 HR 负责人汇报时，与她看重的"底层价值"逻辑高度契合。
